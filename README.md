# 🏬 Interactive Mall Map Prototype

## Описание проекта

Это одностраничное приложение визуализирует этаж торгового центра с интерактивными магазинами.  
Проект построен с использованием:

- **JavaScript (ES6+)**
- **Three.js / WebGL**
- **CSS2DRenderer** для подписей магазинов (или спрайтов при оптимизации)
- **GSAP** для плавных анимаций камеры
- **OrbitControls** для 3D-навигации

### Основные функции:

1. **Динамическое построение сцены**
    - 100 объектов (магазины/зоны) размещаются на сетке с учётом размеров.
    - Каждый объект представлен прямоугольным параллелепипедом.
    - Над объектами отображаются подписи с названием магазина.

2. **Переключение режимов 2D / 3D**
    - **3D**: перспективная камера с объемными объектами.
    - **2D**: ортографическая камера, вид сверху, объекты прижаты к полу.
    - Плавное переключение с анимацией камеры и высоты объектов.

3. **Интерактивность**
    - Подсветка объекта при наведении.
    - Выбор объекта кликом и вывод информации в консоль (`ID`, `Name`, `Position`).

---

## Архитектура

- **scene/** — инициализация Three.js, создание камер и света.
- **data/** — массив `placesData` с описанием магазинов.
- **interaction/** — Raycaster и переключение режимов 2D/3D.
- **scene/createShops.js** — генерация объектов и спрайтов подписей.

Клик по объекту — вывод информации в консоль.


## Ответы на вопросы из раздела "Архитектура и оптимизация"

1. Оптимизация рендеринга текстовых подписей при большом количестве объектов

Вместо DOM-элементов CSS2DObject использовать Sprite с canvas-текстурой.

Canvas создается один раз при генерации объекта, спрайт ориентируется на камеру.


2. Обработка кликов при объединении геометрий

Хранить bounding boxes для интерактивных объектов.

Использовать BVH/Octree для ускорения поиска под курсором.

Raycaster проверяет только интерактивные объекты.

3. Структура данных для быстрого поиска

По ID: Map<string, Shop> — мгновенный доступ.

По координатам: Spatial Hash или Quadtree.

Каждый объект хранит ссылку на mesh и sprite для быстрого изменения состояния.

## Как работает алгоритм:
В функции createShops объекты (магазины) размещаются на сетки (grid) с фиксированным шагом, чтобы гарантировать отсутствие пересечений.



Определяем размер сетки:

const gridSize = 10  // количество объектов в одном ряду
const spacing = 8    // расстояние между объектами

Вычисляем позицию по рядам и колонкам для каждого объекта:

const row = Math.floor(i / gridSize)
const col = i % gridSize

row — индекс текущего ряда (по оси Z).

col — индекс колонки (по оси X).

Размещаем объект на сцене с учётом отступа:

mesh.position.set(
col * spacing,  // X
1,              // Y (высота объекта фиксирована)
row * spacing   // Z
)

Каждый объект занимает свою ячейку сетки.

Фиксированный spacing гарантирует, что объекты не перекрываются, даже если их ширина/глубина немного различаются.

Итеративно строим ряды и колонки:

Первая строка заполняется объектами с col = 0…gridSize-1.

Когда колонка доходит до gridSize, начинается новый ряд (row + 1).

Таким образом, все 100 объектов распределяются аккуратно в виде сетки.
